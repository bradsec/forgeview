   a) `FileMetadata` struct (derives Serialize):
      - path: String, filename: String, extension: String, size_bytes: u64, modified: String

   b) `open_file_dialog` — uses tauri_plugin_dialog::DialogExt to open a file picker:
      - Filter for supported 3D formats: stl, 3mf, obj, gltf, glb, ply, dae
      - Returns Result<Option<String>, String> — the selected file path or None if cancelled
      - Use the blocking file dialog API (dialog().file().blocking_pick_file()) since Tauri v2 dialog plugin provides this

   c) `read_file_bytes` — reads a file at the given path using std::fs::read():
      - Takes path: String
      - Returns Result<Vec<u8>, String>
      - Map errors with .map_err(|e| e.to_string())

   d) `get_file_metadata` — reads file metadata using std::fs::metadata():
      - Takes path: String
      - Extracts filename from path using std::path::Path
      - Returns FileMetadata with size, extension, modified time (formatted as ISO string)
      - Map SystemTime to a human-readable or ISO format string

   e) `get_recent_files` — returns an empty Vec<String> for now (placeholder):
      - Will be implemented with persistence in a future phase

4. Update src-tauri/src/lib.rs:
   - Add `mod commands;` declaration
   - Register plugins: `.plugin(tauri_plugin_dialog::init())` and `.plugin(tauri_plugin_fs::init())`
   - Register command handlers: `.invoke_handler(tauri::generate_handler![commands::file_ops::open_file_dialog, commands::file_ops::read_file_bytes, commands::file_ops::get_file_metadata, commands::file_ops::get_recent_files])`
   - Keep the existing run() function structure from the template

5. Create src-tauri/capabilities/default.json (the directory may need to be created):
   ```json
   {
     "$schema": "../gen/schemas/desktop-schema.json",
     "identifier": "default",
     "description": "Capability for the main window",
     "windows": ["main"],
     "permissions": [
       "core:default",
       "dialog:allow-open",
       "fs:allow-read-file",

       "fs:scope-home-recursive",
       "fs:scope-desktop-recursive",
       "fs:scope-download-recursive",
       "fs:scope-document-recursive"
     ]
   }
   ```

CRITICAL: Permissions go in capabilities/default.json, NOT in tauri.conf.json. This is Tauri v2 — not v1.
CRITICAL: The dialog plugin in Tauri v2 uses tauri_plugin_dialog — check the actual API.
